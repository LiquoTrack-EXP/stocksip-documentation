### 7.4.3. Alerting Pipeline Components

El pipeline de alertas de StockSip permite notificar automáticamente al equipo de desarrollo cuando se detectan incidencias relevantes en cualquiera de los componentes del sistema (Landing Page, Web Application, Web Services API, Aplicación Móvil Android y Aplicación Multiplataforma Flutter).

**Alertas configuradas:**

* Fallos en el despliegue de la aplicación Backend en Azure Web App (Azure Application Insights / Azure Monitor Alerts).
* Tiempo de respuesta superior a 2 segundos en los endpoints principales de la API (gestión de inventario, almacenes y productos).
* Caídas o errores de conexión en la base de datos no relacional Azure Cosmos DB (MongoDB).
* Errores en el proceso de build o despliegue de la Landing Page y la Web Application en Vercel (Vercel Deployment Alerts).
* Fallos en la distribución de nuevas versiones de la aplicación Android a través de Firebase App Distribution.
* Errores en la generación o subida de releases de las aplicaciones móviles (Android nativa y Flutter) mediante GitHub Releases.
* Fallos en las transacciones procesadas a través de Mercado Pago durante el flujo de suscripciones.
* Errores en el servicio de almacenamiento de imágenes en Cloudinary (por ejemplo, al exceder el límite de tamaño permitido por el plan gratuito).

**Prácticas aplicadas:**

* Definición de umbrales de rendimiento en los servicios de Azure para detectar degradaciones en la API antes de que afecten al usuario final.
* Monitoreo del estado de los recursos de Azure (Web App y Cosmos DB) desde el portal de Azure, permitiendo una respuesta temprana ante caídas del servicio.
* Revisión periódica de los logs de despliegue en Vercel para identificar errores de compilación antes de que lleguen a producción.
