## 7.4. Continuous Monitoring

### 7.4.1. Tools and Practices

A continuación se describen las herramientas y prácticas que el equipo de StockSip emplea para llevar a cabo un monitoreo continuo y eficaz de la plataforma, considerando las particularidades de una solución orientada a la gestión de inventarios de licorerías con componentes web, móvil y backend desplegados en la nube.

- **Pruebas de Carga y Estrés:** Gatling permite al equipo de StockSip simular múltiples usuarios accediendo simultáneamente a la plataforma, reproduciendo escenarios de alta demanda como el registro masivo de productos, la consulta de inventarios en tiempo real o el procesamiento de pedidos entre licorerías y proveedores. Sus reportes detallados de rendimiento facilitan la identificación de cuellos de botella tanto en la aplicación web desplegada en Vercel como en el backend alojado en Azure Web App.

<img src="https://imgur.com/zZeDnFt.png" />

- **Monitoreo de Experiencia del Usuario:** Microsoft Clarity se integra en la aplicación web de StockSip para registrar mapas de calor y grabaciones de sesiones de usuarios reales, permitiendo al equipo comprender cómo los dueños de licorerías y proveedores interactúan con funcionalidades clave como la gestión de almacenes, el control de stock y las guías de conservación. Esta información orienta mejoras directas en la usabilidad de la plataforma.

<img src="https://imgur.com/rq91ftp.png" />

- **Supervisión de APIs:** Dado que StockSip expone una API RESTful desplegada en Azure Web App e integra servicios externos como Cloudinary, Google Services y Mercado Pago, el equipo utiliza Postman para verificar la disponibilidad y el correcto funcionamiento de los endpoints críticos. Adicionalmente, Pingdom permite rastrear en tiempo real la disponibilidad del backend, notificando al equipo ante cualquier interrupción que pueda afectar la gestión de inventarios o el procesamiento de órdenes.

<img src="https://imgur.com/nJ6gNj7.png" />

- **Auditorías de Calidad Web:** Google Lighthouse se emplea para auditar periódicamente la landing page y la aplicación web de StockSip, evaluando métricas de accesibilidad, rendimiento y SEO, en línea con las directrices definidas en las Web Style Guidelines del proyecto. Estas auditorías son especialmente relevantes dado que la plataforma debe ser accesible para dueños de licorerías con distintos niveles de experiencia tecnológica y desde diversos tipos de dispositivos.
