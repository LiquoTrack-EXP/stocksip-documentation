## 7.3. Continuous Deployment

El objetivo de Continuous Deployment (CD) es que los cambios aprobados en el código pasen automáticamente desde el desarrollo hasta la producción, garantizando que cada nueva versión sea entregada sin intervención manual, siempre y cuando pase todas las validaciones establecidas por el equipo.

### 7.3.1. Tools and Practices

En este apartado se detallan las herramientas y prácticas que aseguran un despliegue automatizado y confiable en producción.

**Tools:**

- **GitHub Actions:** Para automatizar el pipeline de CI/CD. Esta herramienta permite configurar workflows que incluyen la ejecución de pruebas y el despliegue automático a diferentes entornos (desarrollo, staging, producción), integrándose directamente con los repositorios alojados en la organización **LiquoTrack-EXP** en GitHub.

- **Azure App Service:** Como plataforma de despliegue para la aplicación backend desarrollada en **ASP.NET Core con C#**. Azure Web App permite publicar el proyecto directamente desde el IDE Rider mediante el plugin **Azure Toolkit for Rider**, gestionando la infraestructura en la nube de forma automática. Además, permite configurar las variables de entorno necesarias desde la sección **Configuration** del servicio.

- **Azure Cosmos DB:** Como servicio de base de datos NoSQL en la nube. Cosmos DB permite desplegar y gestionar el clúster de base de datos compatible con MongoDB, configurando reglas de firewall y proporcionando la cadena de conexión que utiliza la aplicación backend para acceder a los datos en producción.

- **Vercel:** Para el despliegue continuo tanto de la **Landing Page** (sitio estático en HTML, CSS y JavaScript) como de la **Aplicación Web Frontend** (desarrollada con Vue y Vite). Vercel se integra directamente con los repositorios de GitHub, detectando automáticamente cada nuevo commit en la rama principal y ejecutando el proceso de construcción y publicación. Las variables de entorno como `VITE_API_BASE_URL` se configuran directamente en la plataforma para los entornos de **Production** y **Preview**.

- **Firebase App Distribution:** Para la distribución de la **aplicación móvil nativa Android** (Kotlin + Jetpack Compose). Firebase permite subir el APK firmado generado desde Android Studio y distribuirlo al equipo de pruebas de forma controlada.

- **GitHub Releases:** Como canal adicional de distribución para las aplicaciones móviles, tanto la **nativa en Android** como la **multiplataforma en Flutter**. Permite publicar versiones estables del APK directamente desde el repositorio privado correspondiente, enlazando la URL de descarga desde la landing page.

**Practices:**

- **Feature Branching:** Se utiliza una estrategia de ramas basada en GitFlow, donde los desarrolladores trabajan en nuevas funcionalidades dentro de ramas de tipo `feature/`. Una vez completadas y revisadas, estas ramas se fusionan a la rama `develop`, que centraliza los cambios antes de pasar a producción.

- **Commit-based Deployment:** Cada vez que se realiza un commit en la rama principal de los repositorios conectados a Vercel, el proceso de construcción y despliegue se activa de forma automática, manteniendo el flujo de trabajo ágil y reduciendo la intervención manual.

- **Release Branching:** Antes de publicar una nueva versión de las aplicaciones móviles, los cambios de la rama `develop` se trasladan a una rama de tipo `release/` (por ejemplo, `release/2.0.0`), desde la cual se genera el APK firmado y se sube tanto a Firebase App Distribution como a GitHub Releases.

- **Semantic Versioning:** Cada versión lanzada sigue la convención de versionado semántico (`MAJOR.MINOR.PATCH`), donde el número principal incrementa al finalizar cada sprint, el secundario al agregar funcionalidades menores, y el terciario al corregir errores puntuales.

- **Rollback manual por ramas:** En caso de detectar errores críticos en producción tras un despliegue, el equipo puede revertir rápidamente al commit anterior desde GitHub o reimplantar una versión anterior del APK desde GitHub Releases, garantizando estabilidad y una recuperación ágil ante fallos.
