### 7.2.2. Stages Deployment Pipeline Components

Esta sección detalla las etapas del pipeline de Entrega Continua (CD) que aseguran un flujo de despliegue automatizado y confiable hacia los diferentes entornos de la solución.  

- **Source Stage:** Etapa inicial donde GitHub Actions detecta cambios en las ramas principales o de lanzamiento (release/) y recupera el código fuente más reciente.

- **Build & Validation Stage:** Se ejecutan los procesos de compilación y las pruebas unitarias. Si se detecta alguna falla o error, el pipeline se detiene automáticamente para proteger la integridad del entorno.  

- **Configuration & Secret Management Stage:** Inyección dinámica de variables de entorno y credenciales sensibles (como cadenas de conexión a Azure Cosmos DB) almacenadas en los GitHub Secrets, evitando su exposición en el código.  

- **Deployment Execution Stage:**
    - **Cloud Hosting (Azure App Service):** El paquete del backend se despliega automáticamente en la Azure Web App, configurando la infraestructura necesaria para su ejecución.  

    - **Global Distribution (Vercel):** El frontend compilado se publica en la red de Vercel, invalidando la caché anterior para que los usuarios reciban la versión actualizada.  

- **Verification & Monitoring Stage:**
    - **Smoke Testing:** Una vez completado el despliegue, se verifica la disponibilidad del servicio accediendo a la interfaz de Swagger del backend y a la URL pública del frontend.  

    - **Release Management:** Se genera una nueva etiqueta de versión siguiendo la convención de Semantic Versioning en el repositorio de GitHub para el seguimiento histórico de la entrega.
