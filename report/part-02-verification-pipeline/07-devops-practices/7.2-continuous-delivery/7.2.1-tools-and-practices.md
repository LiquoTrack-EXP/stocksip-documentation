## 7.2. Continuous Delivery

Continuous Delivery (CD) es una praxis de desarrollo que permite que los equipos de desarrollo entreguen software de forma continua a los clientes. 

El objetivo de CD es reducir el tiempo entre la escritura del código y su despliegue en producción, asegurando que el software esté siempre en un estado desplegable.

### 7.2.1. Tools and Practices

#### **Tools**

- **GitHub Actions**: Es una plataforma de integración continua y entrega continua (CI/CD) que permite a los desarrolladores automatizar sus flujos de trabajo de desarrollo de software directamente desde sus repositorios de GitHub. Con GitHub Actions, los desarrolladores pueden crear flujos de trabajo personalizados para compilar, probar y desplegar su código cada vez que se realice un cambio en el repositorio.



- **GitLab CI/CD**: s una plataforma que permite automatizar las etapas de integración, prueba y despliegue de aplicaciones dentro de GitLab. Facilita la ejecución de pipelines para compilar proyectos, ejecutar pruebas automáticas y desplegar versiones en distintos entornos de manera eficiente. 



- **Jenkins**: Es una herramienta de automatización de código abierto utilizada para implementar procesos de integración y entrega continua. Jenkins permite crear pipelines personalizados para automatizar tareas como compilación, pruebas y despliegue de aplicaciones.


#### **Practices**


- **Automatización de despliegues**: Permite publicar nuevas versiones del software de manera rápida y consistente mediante pipelines automatizados.


- **Pruebas automatizadas**: Consiste en ejecutar pruebas funcionales, unitarias o de integración automáticamente antes de desplegar una nueva versión.


- **Integración continua**: Se realizan cambios frecuentes en el repositorio para detectar errores tempranamente y mantener la estabilidad del sistema.


- **Uso de pipelines de entrega**: Se establecen etapas automatizadas para compilar, validar y desplegar el software en diferentes entornos.


- **Versiones del software**: Permite controlar y gestionar las distintas versiones liberadas de la aplicación.


- **Monitoreo posterior al despliegue**: Consiste en supervisar el comportamiento del sistema después de cada entrega para detectar posibles fallos o problemas de rendimiento.
