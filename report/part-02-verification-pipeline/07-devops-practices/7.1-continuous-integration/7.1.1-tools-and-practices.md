# Capítulo 7: DevOps Practices

## 7.1. Continuous Integration

### 7.1.1. Tools and Practices

En esta sección se detallan los recursos y metodologías aplicadas para la integración constante de cambios en los repositorios de la organización LiquoTrack-EXP.  

- **Tools:**

    - **GitHub Actions:** Plataforma principal para la automatización de flujos de trabajo de CI/CD integrada directamente en los repositorios de GitHub. Permite la ejecución de pipelines de compilación y pruebas ante cada evento de código.  


    - **NuGet:** Gestor de paquetes para el ecosistema .NET que permite la resolución y restauración automática de dependencias del backend.  


    - **NPM (Node Package Manager):** Herramienta utilizada para la gestión de librerías y dependencias del frontend desarrollado en Vue.  

- **Practices:**


    - **Frecuent Commits:** Los desarrolladores integran sus cambios en el repositorio de forma constante para reducir conflictos y detectar errores de integración tempranamente.  


    - **Automated Build:** Cada vez que se realiza un push o Pull Request, el pipeline compila automáticamente el código fuente para asegurar que no existan errores sintácticos o de configuración.  


    - **Pull Request Reviews:** Antes de fusionar cambios a la rama principal, se requiere una revisión por pares para garantizar el cumplimiento de estándares de calidad y evitar la introducción de fallos técnicos.  


    - **Self-Testing Code:** Los desarrolladores son responsables de incluir pruebas unitarias en sus contribuciones para validar la funcionalidad del nuevo código de forma aislada.
