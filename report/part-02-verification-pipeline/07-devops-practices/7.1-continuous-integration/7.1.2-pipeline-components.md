# 7.1.2. Build & Test Suite Pipeline Components
En este apartado se describen los componentes técnicos integrados en el pipeline de Integración Continua (CI) a través de GitHub Actions para garantizar la calidad del código antes de su integración.  


- **CI Automation Engine (GitHub Actions):** Orquestador que ejecuta los flujos de trabajo (workflows) definidos en archivos YAML, activándose automáticamente ante cada push en las ramas de desarrollo.  
- **Source Code Management System (GitHub):** Actúa como la fuente de verdad donde se centraliza el código y se gestionan las versiones mediante ramas de tipo feature/ y develop.  

- **Build Tools & Environments:**:

    - **Backend (SDK de .NET):** Utilizado para restaurar las dependencias de NuGet y compilar el proyecto de la API desarrollado en C#.  

    - **Frontend (Node.js & Vite):** Entorno encargado de procesar y compilar la aplicación web desarrollada en Vue para generar los archivos estáticos de producción.  

- **Automated Testing Suite:**
    - **Unit Tests (MSTest / NUnit):** Pruebas automatizadas que validan de forma aislada la lógica de los controladores y servicios del backend en ASP.NET Core.  

    - **Static Code Analysis (Linters):** Herramientas que analizan el código del frontend para asegurar el cumplimiento de las reglas de estilo y buenas prácticas antes de la construcción.  
    
    - **Artifact Storage:** Almacenamiento temporal de los binarios y paquetes generados que han superado las pruebas, dejándolos disponibles para la siguiente etapa de despliegue.
