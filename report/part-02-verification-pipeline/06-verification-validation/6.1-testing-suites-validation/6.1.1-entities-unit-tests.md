# Part 2: Verification, Validation and Pipelines

En esta segunda parte se abordan los procesos fundamentales que garantizan la calidad del sistema desarrollado. Se presentan estrategias de verificación y validación orientadas a asegurar que el software cumpla con los requisitos funcionales. Además, se describen los pipelines de integración y despliegue que permiten automatizar pruebas, reducir errores en producción y mejorar la confiabilidad del ciclo de desarrollo.

# Capítulo 6: Product Verification & Validation

Este capítulo se enfoca en las prácticas de verificación y validación aplicadas al producto de software. Se describen las estrategias de prueba utilizadas para asegurar la correcta implementación de las entidades centrales del dominio, así como la consistencia del comportamiento del sistema en distintos escenarios. El objetivo principal es garantizar que el producto no solo esté correctamente construido, sino que también responda adecuadamente a las necesidades del negocio.

## 6.1. Testing Suites & Validation

En esta sección se presentan los conjuntos de pruebas implementados para validar el comportamiento del sistema a distintos niveles. Se incluyen pruebas unitarias, de integración y validaciones específicas del dominio, organizadas dentro de suites de testing que permiten ejecutar escenarios de forma automatizada y repetible. Estas herramientas aseguran la estabilidad del sistema ante cambios y facilitan la detección temprana de errores.

### 6.1.1. Core Entities Unit Tests

Esta subsección describe las pruebas unitarias aplicadas a las entidades centrales del dominio. El objetivo es validar que cada entidad cumpla correctamente sus reglas de negocio, invariantes y comportamientos esperados de forma aislada. Estas pruebas permiten asegurar la integridad del modelo de dominio, garantizando que las operaciones fundamentales se ejecuten de manera consistente sin depender de componentes externos.
