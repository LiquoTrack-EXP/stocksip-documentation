### 6.1.3. Core Behavior-Driven Development

#### Prueba BDD 1: Registro de Usuarios

En esta primera prueba se detalla la validación del proceso de creación de cuentas para nuevos usuarios en la plataforma StockSip. La prueba se centra en la Historia de Usuario US001, la cual permite a los visitantes registrar su empresa y datos personales para acceder a las herramientas de gestión de inventario. El objetivo es verificar que el sistema procese correctamente las solicitudes legítimas y bloquee aquellas que carecen de información obligatoria.

El análisis de esta prueba arroja que el flujo de registro es robusto y cumple con los requisitos de negocio. Se observa que el sistema integra correctamente la lógica de persistencia y validación de datos. La implementación técnica mediante una API REST asegura que los campos obligatorios como el correo y el nombre de la empresa sean validados antes de la creación del usuario. Además, el uso de escenarios de prueba parametrizados permite verificar el comportamiento del sistema ante múltiples perfiles de usuario de manera eficiente.

#### Prueba BDD 2:  Inicio de Sesión 

Esta segunda prueba se aborda la validación del sistema de autenticación para los empleados y dueños de negocio. Basada en la Historia de Usuario US011, la prueba busca garantizar que solo los usuarios con credenciales activas y válidas puedan acceder a las áreas protegidas de la plataforma. Se evalúa tanto el acceso exitoso como la respuesta del sistema ante intentos de ingreso con datos erróneos o no registrados.

A través de esta prueba, se determina que el mecanismo de autenticación responde de forma segura y consistente. El sistema logra identificar correctamente las cuentas activas y generar los tokens de acceso necesarios para la navegación. Un aspecto clave destacado en el análisis es el manejo de la seguridad: el sistema deniega el acceso con un código de error de desautorización cuando las credenciales no coinciden, protegiendo la integridad de la información del negocio. La automatización de este flujo previene regresiones en el sistema de seguridad en futuras actualizaciones.
