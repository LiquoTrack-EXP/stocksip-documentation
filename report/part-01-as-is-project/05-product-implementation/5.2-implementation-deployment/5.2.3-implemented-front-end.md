### 5.2.3. Implemented Frontend Web Application Evidence

Para el despliegue de esta aplicación, se utilizó Vercel, una plataforma que permite alojar aplicaciones web de manera sencilla y eficiente, con integración directa a repositorios de GitHub.

Link a la aplicación: [https://stocksip-front-end-application-navy.vercel.app/](https://stocksip-front-end-application-navy.vercel.app/)

* Partimos de la rama master del repositorio de la aplicación frontend con todos los cambios realizados para el sprint actual.

<p align="center">
  <img src="https://imgur.com/V9gGY1D.png"/>
</p>

* Luego, ingresamos a la plataforma Vercel e iniciamos sesión con nuestra cuenta. Una vez dentro, seleccionamos la opción de importar un repositorio de GitHub. Se nos presentó la lista de repositorios disponibles bajo la organización **LiquoTrack-EXP**, desde donde seleccionamos el repositorio **stocksip-front-end-application** haciendo clic en el botón **Import**.

<p align="center">
  <img src="https://imgur.com/fWcHboG.png"/>
</p>

* A continuación, Vercel nos presentó la pantalla de configuración del nuevo proyecto. Verificamos que el nombre del proyecto fuera **stocksip-front-end-application**, que el equipo seleccionado fuera **matias' projects** y que el preset de la aplicación fuera **Vite**, correspondiente al framework utilizado en el proyecto. El directorio raíz se mantuvo en `./`.

<p align="center">
  <img src="https://imgur.com/GdTCRfF.png"/>
</p>

* Luego, expandimos la sección de **Environment Variables** para configurar las variables de entorno necesarias para la aplicación. Se nos presentó el formulario con los campos **Key**, **Value** y el ambiente de destino **Production and Preview**.

<p align="center">
  <img src="https://imgur.com/D0feh5S.png"/>
</p>

* A continuación, agregamos la variable de entorno **VITE_API_BASE_URL** con el valor correspondiente a la URL base de la API del backend. Esta variable es requerida por la aplicación para establecer la conexión con los servicios del servidor, de igual manera continuamos agregando las demas variables necesarias.

<p align="center">
  <img src="https://imgur.com/UrDfhTF.png"/>
</p>

* Una vez completada la configuración, hicimos clic en el botón **Deploy**. Vercel inició el proceso de despliegue, mostrando los **Build Logs** en tiempo real con el estado **Deploying outputs**. El proceso tomó aproximadamente 12 segundos en completarse.

<p align="center">
  <img src="https://imgur.com/w7nD9tC.png"/>
</p>

* Finalmente, tras la ejecución exitosa del despliegue del Frontend a través de Vercel, se muestra la evidencia del resultado: la aplicación **StockSip** desplegada y accesible, mostrando correctamente la pantalla de inicio de sesión con las opciones de ingreso por email, Google y GitHub.
