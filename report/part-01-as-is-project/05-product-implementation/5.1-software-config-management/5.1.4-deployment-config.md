### 5.1.4. Software Deployment Configuration

En esta sección se describen únicamente las **configuraciones de despliegue** definidas para cada producto de software del proyecto. No se detallan pasos operativos ni procedimientos de publicación, ya que el objetivo de esta sección es evidenciar la infraestructura, servicios y parámetros utilizados para habilitar cada entorno de ejecución.

### Despliegue del Landing Page con Vercel ###

* **Paso 1: Importar el repositorio en Vercel**
  Como primer paso, se ingresó a la plataforma Vercel y se inició sesión con la cuenta del equipo. Una vez dentro, se seleccionó la opción **Add New** y luego **Project** para importar el repositorio **stocksip-landing-page** desde la organización **LiquoTrack-EXP** en GitHub.

<p align="center">
  <img src="https://imgur.com/XOKJCKY.png"/>
</p>

<p align="center">
  <img src="https://imgur.com/UxZEdhC.png"/>
</p>

* **Paso 2: Configuración del proyecto**
  Se verificó que el nombre del proyecto fuera **stocksip-landing-page-3wct** y que el preset de la aplicación fuera **Other**, correspondiente a la naturaleza estática de la landing page. El directorio raíz se mantuvo en `./`.

<p align="center">
  <img src="https://imgur.com/HsgR97j.png"/>
</p>

* **Paso 3: Selección del directorio raíz**
  Dado que el código fuente se encuentra dentro de la carpeta `src`, se expandió la sección **Root Directory** y se seleccionó el subdirectorio `src` como directorio raíz del proyecto.

<p align="center">
  <img src="https://imgur.com/z1TJYn7.png"/>
</p>

* **Paso 4: Despliegue**
  Una vez completada la configuración, se hizo clic en el botón **Deploy**. Vercel procesó el repositorio exitosamente y el estado del deployment cambió a **Ready**, confirmando que la landing page fue desplegada correctamente bajo el dominio **stocksip-landing-page.vercel.app**.

<p align="center">
  <img src="https://imgur.com/74JTuiW.png"/>
</p>

* **Paso 5: Verificación del despliegue**
  Finalmente, la landing page de **StockSip** se encuentra accesible y muestra correctamente el hero section, las métricas clave y la sección de Misión, Visión y Valores.

<p align="center">
  <img src="https://imgur.com/JRFyLiR.png"/>
</p>

### Despliegue de la Base de Datos NoSQL con Microsoft Azure (Azure Cosmos DB) ###

A continuación, se describen los pasos para desplegar la base de datos NoSQL utilizando Azure Cosmos DB:

* **Paso 1: Creación del recurso en el portal de Azure**
  Como primer paso, se inició sesión en la cuenta de Azure y se seleccionó "**Crear un recurso**".

<img src="https://i.imgur.com/KrfMsDr.png" alt="mongo-db-step-1" style="width: 100%; height: auto;"/>

* **Paso 2: Selección del servicio Azure Cosmos DB**
  Como segundo paso, se buscó "**Azure Cosmos DB**" y se seleccionó la opción para crear una nueva cuenta. Se eligió "**Azure Cosmos DB for MongoDB**" por su capa de compatibilidad con herramientas, controladores y SDKs de MongoDB. Dentro de esta opción, se seleccionó el tipo de recurso "**vCore cluster**" por ser el más adecuado para la mayoría de las aplicaciones.

<img src="https://i.imgur.com/L6HDBdb.png" alt="mongo-db-step-2" style="width: 100%; height: auto;"/>

<img src="https://i.imgur.com/s7zIMLW.png" alt="mongo-db-step-3" style="width: 100%; height: auto;"/>

<img src="https://i.imgur.com/CIOz4PT.png" alt="mongo-db-step-4" style="width: 100%; height: auto;"/>

* **Paso 3: Creación de la base de datos con MongoDB**
  Como tercer paso, se completaron los datos solicitados, como el nombre de la cuenta, el grupo de recursos, la ubicación y las opciones de rendimiento, para posteriormente crear el recurso.

<img src="https://i.imgur.com/0VeP6OE.png" alt="mongo-db-step-4" style="width: 100%; height: auto;"/>

* **Paso 4: Habilitar puertos para acceder localmente**
  Como cuarto paso, se habilitaron todos los puertos para permitir el acceso a la base de datos de forma local desde MongoDB Compass para todos los miembros del equipo, configurando las reglas de firewall en el portal de Azure.

<img src="https://i.imgur.com/pwKK8vx.png" alt="mongo-db-step-5" style="width: 100%; height: auto;"/>

* **Paso 5: Acceso desde MongoDB Compass**
  Finalmente, se accedió a la base de datos desde MongoDB Compass utilizando la cadena de conexión proporcionada en el portal de Azure, reemplazando "**< password >**" por la contraseña creada en el paso 3.

<img src="https://i.imgur.com/mkI79eU.png" alt="mongo-db-step-6" style="width: 100%; height: auto;"/>

<img src="https://i.imgur.com/L4OoDFA.png" alt="mongo-db-step-7" style="width: 100%; height: auto;"/>
