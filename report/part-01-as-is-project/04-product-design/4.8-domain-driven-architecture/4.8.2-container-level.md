##### 4.8.2. Software Architecture Container Level Diagrams

El diagrama de contenedores de la arquitectura de software proporciona una visión de alto nivel de los principales contenedores (aplicaciones, servicios, bases de datos, etc.) que componen el sistema y cómo interactúan entre sí.

Para StockSip, el diagrama de contenedores incluye los siguientes contenedores principales:

<img src="https://i.imgur.com/fKft7m2.png" alt="container-diagram" style="width: 100%; height: auto;"/>


- **Contenedores**:
  - **StockSip Landing Page**: Una página web estática que proporciona información sobre StockSip y permite a los visitantes registrarse o iniciar sesión. Hecha con HTML5, CSS3 y JavaScript.
  - **StockSip Web Application**: El servidor web que entrega el Client-Side Application (SPA), además de manejar las solicitudes HTTP de los usuarios.
  - **StockSip Platform Web App**: Una aplicación web que interactúa con los usuarios finales, permitiéndoles gestionar el inventario, realizar ventas y comunicarse con proveedores. Hecha con JavaScript y Vue.Js.
  - **StockSip Android Application**: Una aplicación móvil nativa para dispositivos Android que será usada por los dueños de licorerías. Hecha con Kotlin y Jetpack Compose.
  - **Mobile SQL Database**: Una base de datos local que almacena información relevante para la aplicación móvil, como pedidos y direcciones de entrega. Hecha con SQLite.
  - **StockSip Multiplatform Mobile Application**: Una aplicación móvil multiplataforma que será usada por los proveedores de licorerías. Hecha con Flutter y Dart.
  - **StockSip Sever Side App**: Una aplicación backend que maneja la lógica de negocio, la gestión de datos y las interacciones con otros sistemas. Hecha con C# y ASP.NET CORE.
  - **Stocksip Database**: Una base de datos no relacional que almacena la información del sistema, como usuarios, productos, ventas y proveedores. Hecha con MongoDB.
