### 4.6.4. Web Applications Userflow Diagrams

Un user flow o trayectoria del usuario es un diagrama que consiste en mostrar el recorrido del usuario representado mediante un diagrama de flujo, el cual indica el camino que debe seguir para cumplir con un objetivo específico dentro de la aplicación. Además, el user flow determina los pasos necesarios para completar una experiencia digital satisfactoria, contemplando tanto la ruta esperada (happy path) como las rutas alternas (unhappy paths) que surgen ante errores o decisiones inesperadas del usuario.

#### User Goal 1: Usuario desea registrarse en la aplicación
 
**Happy Path**
 
En esta ruta esperada, el flujo concentra el proceso de creación de una cuenta en la aplicación StockSip. El usuario es recibido por la pantalla principal y accede a la sección de registro, donde completa los campos requeridos (nombre completo, correo electrónico, número de teléfono y contraseña). Al confirmar el registro, el sistema crea la cuenta y redirige al usuario a la pantalla de inicio de sesión para que pueda acceder a la plataforma.

</p>

**Unhappy Paths**
 
Estas rutas alternas consideran los casos en que el usuario ha ingresado información incorrecta o incompleta, como un correo ya registrado, una contraseña que no cumple los requisitos mínimos de seguridad, o campos obligatorios vacíos. En estos casos, el sistema muestra mensajes de validación que indican el error específico y permiten al usuario corregir la información sin perder los datos ya ingresados.

#### User Goal 2: Usuario desea iniciar sesión en la aplicación
 
**Happy Path**
 
En esta ruta esperada, el usuario accede a la pantalla de login de StockSip, ingresa su correo electrónico y contraseña registrados, y selecciona el botón "Sign In". El sistema valida las credenciales correctamente y redirige al usuario al dashboard principal, desde donde puede acceder a todos los módulos de la plataforma.
 
<p align="center">
  <img src="https://imgur.com/fiRFUar.png" alt="userflow_2_happypath"/>
</p>

**Unhappy Paths**
 
Estas rutas alternas contemplan los escenarios en que el usuario ingresa credenciales incorrectas o no reconocidas por el sistema. En caso de error, la plataforma muestra un mensaje de validación indicando que el correo o la contraseña son incorrectos. El usuario puede optar por reintentar el inicio de sesión con los datos correctos o acceder al flujo de recuperación de contraseña mediante el enlace "Forgot password?", que redirige a un proceso de restablecimiento por correo electrónico.
 
<p align="center">
  <img src="https://imgur.com/Hc5CIMh.png" alt="userflow_2_unhappypath"/>
</p>

#### User Goal 3: Usuario desea explorar un catálogo y agregar un activo al carrito
 
**Happy Path**
 
En esta ruta esperada, el usuario navega desde el dashboard hacia la sección "Catálogos", donde visualiza las colecciones de activos disponibles. Selecciona el catálogo de un proveedor de su interés, explora los productos listados y accede al detalle de un ítem específico, donde revisa su precio, unidades disponibles y rendimiento histórico. Finalmente, ajusta la cantidad deseada y pulsa "Add to Cart", momento en que el sistema confirma la adición y el usuario es redirigido al carrito de compras.
 
<p align="center">
  <img src="https://imgur.com/1pVqQlA.png" alt="userflow_3_happypath"/>
</p>

**Unhappy Paths**
 
Estas rutas alternas consideran los casos en que el ítem seleccionado no cuenta con unidades disponibles en stock, situación en la que el sistema deshabilita el botón de añadir al carrito y muestra un indicador de "Out of Stock". También se contempla el caso en que el usuario intenta agregar una cantidad superior al stock disponible, donde el sistema limita automáticamente la cantidad al máximo disponible y notifica al usuario mediante un mensaje de alerta.
 
<p align="center">
  <img src="https://imgur.com/aSRFWtu.png" alt="userflow_3_unhappypath"/>
</p>


#### User Goal 4: Usuario desea transferir inventario entre almacenes
 
**Happy Path**
 
En esta ruta esperada, el usuario accede a la sección "Inventory" y selecciona la acción "Transfer Stock" desde el panel de acciones rápidas. El sistema despliega el formulario de transferencia donde el usuario selecciona el almacén de origen, el activo a transferir, el almacén de destino y la cantidad. Al confirmar con "Execute Transfer", el sistema actualiza el stock en ambas ubicaciones y notifica al usuario con la confirmación de la operación.
 
<p align="center">
  <img src="https://imgur.com/8ZTF9Mj.png" alt="userflow_6_happypath"/>
</p>

**Unhappy Paths**
 
Estas rutas alternas consideran los casos en que la cantidad ingresada para la transferencia supera el stock disponible en el almacén de origen, situación en que el sistema muestra un mensaje de error indicando las unidades máximas transferibles. También se contempla el caso en que el usuario selecciona el mismo almacén como origen y destino, donde el sistema bloquea la operación y solicita seleccionar instalaciones distintas.

</p>
