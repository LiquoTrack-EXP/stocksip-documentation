### 4.1.2. Web Style Guidelines

Esta sección define las pautas visuales y de interfaz para el desarrollo de la aplicación en **Vue**, utilizando **PrimeVue** como biblioteca principal de componentes. El enfoque está centrado en garantizar una experiencia consistente, accesible y optimizada para todos los usuarios, independientemente del dispositivo que utilicen.

**Responsive Design**

La interfaz de StockSip se adapta automáticamente mediante el uso de CSS Grid, Flexbox y utilidades responsivas, junto con breakpoints estándar para diseño adaptable:

- **xs** ≤ 576px (móvil)
- **sm** 577–768px (tablet)
- **md** 769–1024px (laptop)
- **lg** > 1024px (escritorio)

Se emplean clases utilitarias y media queries para estructurar la interfaz de forma fluida y adaptativa.

**Componentes y patrones compatibles**

- **Navbar/Menu**: `<Menubar>` o `<Toolbar>` junto con `<Sidebar>` para la navegación principal. Se integran con rutas usando `router-link`.
- **Cards**: `<Card>` para contenedores de información como productos, alertas o resúmenes.
- **Botones**: `<Button>` con variantes (`label`, `icon`, `severity`, `outlined`, etc.) según el contexto.
- **Notificaciones y Diálogos**: `<Toast>` para mensajes de éxito, error o alerta; `<Dialog>` para mostrar detalles o acciones contextuales. Gestionados mediante servicios de PrimeVue.

**Accesibilidad**

Todos los componentes siguen prácticas de accesibilidad:

- PrimeVue incluye soporte para atributos `aria-*` en componentes relevantes.
- Navegación por teclado compatible (`Tab`, `Enter`, `Space`, `Escape`, etc.).
- Contraste visual conforme a la norma WCAG 2.1 AA.
- Tipografías y layouts adaptables mediante unidades relativas (`rem`/`em`), favoreciendo la escalabilidad en distintos dispositivos.
