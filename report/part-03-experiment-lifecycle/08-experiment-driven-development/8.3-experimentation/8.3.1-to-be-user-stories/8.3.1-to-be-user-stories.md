## 8.3. Experimentation

### 8.3.1 To-Be User Stories

Para StockSip, se han definido las siguientes historias de usuario, que representan las funcionalidades clave que se desean implementar en la plataforma. Cada historia de usuario incluye un título, una descripción del objetivo y los criterios de aceptación que deben cumplirse para considerar la historia como completada.

| US-ID | Título                                  | Descripción                                                                                                                                                                                          | User               | Criterio de Aceptación                                                                                                                                                                                                                                                                                                                                                                                                                                  |
|-------|-----------------------------------------|------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|--------------------|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| US-01 | Visualización de Stock en los catálogos | Como dueño de licorería, quiero ver la disponibilidad de stock actual de los productos del proveedor, para no armar un pedido de artículos agotados y asegurar el abastecimiento real de mi negocio. | Dueño de licorería | **Escenario 1:** Dado que estoy viendo el catálogo de un proveedor, cuando un producto tiene stock disponible, entonces el sistema debe mostrar la cantidad disponible junto al producto. <br> <br> **Escenario 2:** Dado que un producto está agotado, cuando visualizo el catálogo, entonces el sistema debe marcarlo visualmente como "Sin stock" y deshabilitar la opción de agregarlo al carrito.                                                  |
| US-02 | Crear dirección de entrega              | Como dueño de licorería, quiero ingresar y registrar una dirección de entrega al realizar mi pedido, para asegurar que el proveedor despache el producto al lugar correcto.                          | Dueño de licorería | **Escenario 1:** Dado que estoy realizando un pedido, cuando llego al paso de entrega, entonces el sistema debe permitirme ingresar los datos de una nueva dirección (calle, distrito, referencia). <br> <br> **Escenario 2:** Dado que ingreso una nueva dirección, cuando los datos obligatorios están incompletos, entonces el sistema debe impedir continuar y mostrar un mensaje de validación.                                                    |
| US-03 | Explorar catálogos de proveedores       | Como dueño de licorería, quiero visualizar los catálogos publicados por distintos proveedores, para comparar productos y encontrar las mejores opciones de abastecimiento.                           | Dueño de licorería | **Escenario 1:** Dado que accedo a la sección de proveedores, cuando existen catálogos publicados, entonces el sistema debe listarlos con nombre del proveedor, categoría principal y calificación (si existe). <br> <br> **Escenario 2:** Dado que estoy explorando catálogos, cuando aplico un filtro (por ejemplo, categoría o ubicación), entonces el sistema debe actualizar la lista mostrando solo los catálogos que cumplen ese criterio.       |
| US-04 | Comprar productos del carrito           | Como dueño de licorería, quiero confirmar la compra de los productos agregados al carrito, para abastecer mi negocio de manera rápida y organizada.                                                  | Dueño de licorería | **Escenario 1:** Dado que tengo productos agregados al carrito, cuando confirmo la compra, entonces el sistema debe generar un pedido con el detalle de productos, cantidades y monto total. <br> <br> **Escenario 2:** Dado que confirmo la compra, cuando algún producto del carrito quedó sin stock antes de finalizar, entonces el sistema debe notificarme y evitar procesar el pedido con ese ítem hasta que se actualice el carrito.             |
| US-05 | Publicar catálogo                       | Como proveedor de licorería, quiero publicar mi catálogo de productos, para que las licorerías puedan visualizar y evaluar mis ofertas comerciales.                                                  | Proveedor          | **Escenario 1:** Dado que soy un proveedor autenticado, cuando completo los datos obligatorios del catálogo (productos, precios, stock), entonces el sistema debe publicarlo y hacerlo visible para las licorerías. <br> <br> **Escenario 2:** Dado que intento publicar un catálogo, cuando falta información obligatoria (por ejemplo, precio o imagen del producto), entonces el sistema debe impedir la publicación y señalar los campos faltantes. |
| US-06 | Editar catálogo                         | Como proveedor de licorería, quiero actualizar la información de mi catálogo, para mantener los precios, stock y productos disponibles siempre actualizados.                                         | Proveedor          | **Escenario 1:** Dado que tengo un catálogo publicado, cuando modifico el precio o stock de un producto, entonces el sistema debe guardar los cambios y reflejarlos de inmediato en la vista de las licorerías. <br> <br> **Escenario 2:** Dado que edito mi catálogo, cuando elimino un producto, entonces el sistema debe retirarlo del catálogo visible sin afectar los pedidos ya confirmados que lo incluyan.                                      |
| US-07 | Visualizar el estado del pedido         | Como dueño de licorería, quiero conocer el estado de mi pedido, para estar informado sobre la entrega y planificar mis operaciones comerciales.                                                      | Dueño de licorería | **Escenario 1:** Dado que tengo un pedido activo, cuando consulto su estado, entonces el sistema debe mostrar el estado actual (por ejemplo: confirmado, en preparación, en camino, entregado). <br> <br> **Escenario 2:** Dado que el estado de mi pedido cambia, cuando el proveedor actualiza dicho estado, entonces el sistema debe notificarme el nuevo estado sin necesidad de recargar manualmente la pantalla.                                  |
