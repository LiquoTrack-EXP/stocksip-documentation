#### 8.3.3.5. Implemented To-Be RESTful API Backend Evidence

Para el desarrollo del experimento, se implementó la funcionalidad de las órdenes de compra y los catálogos de productos mediante el uso del API actual de Stocksip hecho en **.NET Core y C#**. A su vez, se divide el producto internamente en **Bounded Contexts** dependiendo de la capacidad de negocio. Para este caso, las funcionalidades de la orden de compra y los catálogos de compra genera un nuevo dominio que recibe el nombre de **Sales Management**. De esta forma, el API expone controladores (endpoints) que serán consumidos por las aplicaciones web y móvil. Finalmente, la REST API con las implementaciones del experimento se despliega en el servicio de **Azure Websites**.

**Documentación de Servicios del API**

Repositorio: [https://github.com/LiquoTrack-EXP/stocksip-platform-api](https://github.com/LiquoTrack-EXP/stocksip-platform-api)
Enlace al API desplegado: [https://stocksip-back-end.azurewebsites.net/swagger/index.html](https://stocksip-back-end.azurewebsites.net/swagger/index.html)
